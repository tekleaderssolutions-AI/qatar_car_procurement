0. PROBLEM STATEMENT & PLATFORM VISION

0.1 The Problem

Qatar's used car market is fragmented, opaque, and inefficient. Buyers
overpay because they have no reliable pricing benchmark. Sellers
underprice because they cannot see real demand signals. Dealers hold
stagnant inventory for months because they buy based on gut feeling
rather than data. No single platform in Qatar combines:

-   Real-time market pricing intelligence based on actual transaction
    data

-   Demand forecasting driven by Google Trends, economic indicators, and
    seasonal patterns

-   Inventory health monitoring with AI-driven stagnation alerts

-   Buyer-seller matching using customer preference and upgrade cycle
    data

-   Macro-economic context (oil prices, expat population, interest
    rates) linked to car market movements

  🔴 Qatar used car market size: ~QAR 2.8 billion annually. No
  intelligent platform exists. Dubizzle and OpenSooq are classifieds
  sites — they show listings but provide zero intelligence about
  pricing, timing, or market direction.

0.2 The Platform Vision

Build an AI-powered used car market intelligence platform for Qatar that
serves three user types simultaneously:

  ------------------------------------------------------------------------
  User Type          Problem We Solve           Key Feature They Need
  ------------------ -------------------------- --------------------------
  Used Car Dealers   Buying wrong stock,        Inventory health AI +
                     stagnant inventory         demand forecasting

  Individual Sellers Don't know what price to   Instant AI pricing with
                     list at                    market context

  Buyers             Can't tell if price is     Price fairness score +
                     fair or market is moving   market timing alerts

  Fleet Managers     Disposing multiple         Bulk valuation + best
                     vehicles efficiently       platform routing

Python Code Skeleton

  import pandas as pd

  import xgboost as xgb

  from sklearn.model_selection import train_test_split

  from sklearn.preprocessing import LabelEncoder

  from sklearn.metrics import mean_absolute_percentage_error

  # Load data

  df = pd.read_excel('historical_sales.xlsx')

  econ = pd.read_excel('Qatar_Economic_Indicators.xlsx',

  sheet_name='Monthly Data')

  # Merge economic context by date

  df['month_key'] = pd.to_datetime(df['sale_date']).dt.to_period('M')

  econ['month_key'] = pd.to_datetime(econ['Date']).dt.to_period('M')

  df = df.merge(econ[['month_key','oil_price_usd_barrel',

  'interest_rate_pct','consumer_confidence_index',

  'is_peak_buying_season']], on='month_key', how='left')

  # Encode categoricals

  cat_cols = ['make','model','trim','color','condition_label',

  'accident_history','service_history']

  for col in cat_cols:

  df[col] = LabelEncoder().fit_transform(df[col].astype(str))

  # Train

  features = ['make','model','trim','age_years','mileage_km','color',

  'condition_label','accident_history','service_history',

  'oil_price_usd_barrel','interest_rate_pct',

  'consumer_confidence_index','is_peak_buying_season']

  X = df[features]

  y = df['final_sale_price_qar']

  X_train,X_test,y_train,y_test = train_test_split(X,y,test_size=0.2)

  model = xgb.XGBRegressor(n_estimators=500, max_depth=6,

  learning_rate=0.05, random_state=42)

  model.fit(X_train, y_train)

  mape = mean_absolute_percentage_error(y_test, model.predict(X_test))

  print(f'MAPE: {mape:.2%}') # Target: < 8%

  model.save_model('price_predictor_v1.json')

2.5 Model 4 — Buyer-Listing Matcher (P1)

How It Works

Matches active buyers (from customers.xlsx) with current inventory
(vehicle_inventory.xlsx) using a similarity score across multiple
preference dimensions.

  Matching Score Formula:

  match_score = (

  model_match * 0.30 +

  budget_fit * 0.25 +

  color_preference * 0.20 +

  feature_match * 0.15 +

  upgrade_timing * 0.10

  )

  budget_fit:

  listed_price <= customer_budget * 1.1 → 100

  listed_price <= customer_budget * 1.2 → 60

  listed_price > customer_budget * 1.2 → 0

  upgrade_timing:

  next_upgrade_date within 30 days → 100

  within 60 days → 75

  within 90 days → 50

  over 90 days → 20

  ✅ Business Value: Pre-sell inventory before it arrives. A match
  score > 80 triggers automatic WhatsApp/SMS to buyer with listing
  details. This eliminates stagnation on pre-committed stock.

6.1 Complete pytrends Script

  # qatar_trends_pipeline.py

  # Run every Monday 6:00 AM Qatar time (UTC+3)

  from pytrends.request import TrendReq

  import pandas as pd

  import sqlite3

  import time

  from datetime import datetime

  import schedule

  KEYWORDS_BATCHES = [

  ['Toyota Land Cruiser Qatar','Toyota Prado Qatar',

  'Toyota Hilux Qatar','Nissan Patrol Qatar','buy car Qatar'],

  ['Lexus LX Qatar','Lexus GX Qatar','Toyota Fortuner Qatar',

  'Toyota Camry Qatar','new car Qatar'],

  ['best SUV Qatar','used car Qatar','car loan Qatar',

  'electric car Qatar','hybrid car Qatar'],

  ['Land Cruiser price Qatar','car finance Qatar',

  'used Land Cruiser Qatar','Cars24 Qatar','Dubizzle Qatar cars'],

  ['Range Rover Qatar','best family car Qatar',

  'Hilux price Qatar','Lexus ES Qatar','TANK 500 Qatar'],

  ]

  def pull_and_store():

  pytrends = TrendReq(hl='en-US', tz=180, timeout=(10,25), retries=3)

  conn = sqlite3.connect('qauto.db')

  results = []

  for batch in KEYWORDS_BATCHES:

  try:

  pytrends.build_payload(batch, cat=47,

  timeframe='today 3-m', geo='QA')

  df = pytrends.interest_over_time()

  if not df.empty:

  df = df.drop(columns=['isPartial'],errors='ignore')

  df = df.reset_index().melt(id_vars='date',

  var_name='keyword', value_name='trend_score')

  df['pulled_at'] = datetime.now()

  results.append(df)

  except Exception as e:

  print(f'Error: {e}')

  time.sleep(12)

  if results:

  final = pd.concat(results, ignore_index=True)

  final.to_sql('google_trends', conn,

  if_exists='append', index=False)

  print(f'Stored {len(final)} rows at {datetime.now()}')

  conn.close()

  schedule.every().monday.at('06:00').do(pull_and_store)

  while True:

  schedule.run_pending()

  time.sleep(60)

10.6 Signal Fusion — Combining [TRENDS] + [SOCIAL]

When [TRENDS] and [SOCIAL] both confirm demand for the same brand/model
in the same week, procurement confidence is highest. Use this SQL join
pattern:

  -- Demand Confidence Score: fused [TRENDS] + [SOCIAL] signal per
  brand/model/week

  SELECT

  gt.brand,

  gt.model_reference,

  gt.week_start_date,

  gt.trend_score AS google_interest,

  gt.demand_signal,

  gt.alert_flag AS trends_alert,

  AVG(sm.sentiment_score) AS social_sentiment,

  SUM(sm.mention_count) AS total_social_mentions,

  COUNT(CASE WHEN sm.trending_flag=1 THEN 1 END) AS viral_platforms,

  -- Composite confidence: 60% search intent + 40% social sentiment

  ROUND((gt.trend_score * 0.6) + (AVG(sm.sentiment_score)*50 * 0.4), 1)
  AS demand_confidence

  FROM google_trends gt

  LEFT JOIN social_media_trends sm

  ON gt.brand = sm.brand

  AND gt.model_reference = sm.model

  AND gt.week_start_date = sm.week_start_date

  WHERE gt.week_start_date >= DATE('now','-8 weeks')

  AND gt.keyword_type IN
  ('brand_search','model_search','luxury_search','pickup_search')

  GROUP BY gt.brand, gt.model_reference, gt.week_start_date

  ORDER BY demand_confidence DESC;
